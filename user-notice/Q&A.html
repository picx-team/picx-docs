Q & A ​

针对用户的一些高频问题，作者的统一回复和解释说明。

Q: PicX 都有哪些使用载体？ ​

A:

  • PicX 主要的载体为 Web 端网站，无需下载，无需安装，在线使用，主打方便快捷，同时支持 PWA（可将网站安装为 Windows 或 macOS 应用）由 作者（@XPoet）维护。
  • 另外，PicX 也提供了 Windows 和 macOS 的桌面应用，但交互界面和操作方式与 Web 端网站不太一致，由 PicX 贡献者 @Zhazhazhu 维护，目前正处于开发中，暂未发布 1.0 版本。

Q: PicX 是否有 Docker 镜像？ ​

A:

  • PicX 一直处于不断维护和迭代中，官方未曾制作和发布 Docker 镜像，任何在 https://hub.docker.com 发布的 PicX Docker 镜像均属于其个人行为，PicX 官方无法保证其安全和稳定，请谨慎使用。
  • PicX 本身处于持续的维护和迭代中，官方始终认为以网站作为使用载体是最佳方式，无需下载，无需安装，在线使用，减少不必要的使用门槛和心智负担，因此后续也几乎不太可能制作和发布 Docker 镜像，望周知。

Q: PicX 是否有 Chrome 浏览器插件？ ​

A:

  • PicX 官方未曾制作和发布 Chrome 浏览器插件，任何在 Chrome 应用商店发布的 PicX 插件均属于其第三方个人行为，PicX 官方无法保证其安全和稳定，请谨慎使用。
  • PicX 本身处于持续的维护和迭代中，官方始终认为以网站作为使用载体是最佳方式，无需下载，无需安装，在线使用，减少不必要的使用门槛和心智负担，因此后续也几乎不太可能制作和发布 Chrome 浏览器插件，一切为官方公告为准。

Q: 如何自行部署 PicX？ ​

A:

  • PicX 本身处于持续的维护和迭代中，不断有旧 Bug 修复和新功能增加，作者本人极其不建议用户额外自行部署一套 PicX 站点，直接使用官方网站 https://picx.xpoet.cn 即可，有任何问题作者都会及时发版更新，省心、省事、省时。
  • 对于用户自行部署 PicX 产生的一切问题，作者不会给予解答。PicX 代码是百分百开源的，丝毫不用担心数据安全，完全没有额外自行部署的必要。

Q: PicX 自定义图片链接规则问题？ ​

A:

  • PicX 自定义图片链接规则的原理是搭建你自己的服务器来代理请求你的 PicX 图床仓库（即你的 GitHub 仓库）的资源，因此自定义图片链接规则是需要填写你自己的服务器域名或 IP 地址。然而每个人使用的服务器和代理请求的实现方案不尽相同，这其中涉及到方方面面的知识点，通过搜索引擎搜索、询问 ChatGPT 等方式均可得到解决，细节问题作者也难于解答。
